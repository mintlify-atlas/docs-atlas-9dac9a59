## OWASP Top 10 2021 Mapping

### A01:2021 - Broken Access Control

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **Insecure Direct Object Reference (IDOR)** - `vulnerable/app.py:95-110`
    - Users can access other users' profiles by manipulating the `id` parameter
    - No authorization checks before displaying sensitive data
  
  - **Missing Function Level Access Control**
    - Insufficient role-based access controls
    - Weak session validation
  
  **Security Impact:**
  - Unauthorized information disclosure
  - Access to other users' personal data
  - Potential privilege escalation
  
  **Mitigation Applied:**
  - Authorization checks in `secure/app.py:146-158`
  - User can only view their own profile unless admin role
  - Proper session validation on all protected routes
</Accordion>

**Severity:** Critical  
**CVSS Score:** 8.2  
**Affected Code:** Profile view endpoint

---

### A02:2021 - Cryptographic Failures

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **Plaintext Password Storage** - `vulnerable/app.py:26`
    - Passwords stored in database without any encryption
    - Direct string comparison in SQL queries
    - Credentials visible in database dumps
  
  - **Sensitive Data Exposure**
    - Password field returned in user queries
    - SQL errors expose database structure
    - Session secrets hardcoded in source
  
  **Security Impact:**
  - Complete compromise if database is breached
  - User credentials stolen and reused on other sites
  - Compliance violations (GDPR, PCI-DSS)
  
  **Mitigation Applied:**
  - Werkzeug password hashing (bcrypt) in `secure/app.py:104`
  - Environment-based secret key management
  - Password field excluded from SELECT queries
  - Secure session configuration
</Accordion>

**Severity:** Critical  
**CVSS Score:** 9.1  
**Affected Code:** Login and registration functions

---

### A03:2021 - Injection

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **SQL Injection** - `vulnerable/app.py:26`
    - User input directly concatenated into SQL queries
    - No input validation or sanitization
    - Authentication bypass via SQL manipulation
  
  **Attack Vectors Shown:**
  ```sql
  -- Authentication Bypass
  admin' OR '1'='1
  
  -- Comment Injection
  admin'-- 
  
  -- Union-based Injection
  ' OR 1=1 --
  ```
  
  **Security Impact:**
  - Complete authentication bypass
  - Unauthorized data access
  - Potential data modification or deletion
  - Database structure disclosure
  
  **Mitigation Applied:**
  - Parameterized queries (prepared statements) in `secure/app.py:55-56`
  - Input validation and length limits
  - Proper error handling without information disclosure
</Accordion>

**Severity:** Critical  
**CVSS Score:** 9.8  
**Affected Code:** Login, profile, and registration endpoints

---

### A04:2021 - Insecure Design

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **Security Misconfiguration**
    - Debug mode enabled in production
    - Hardcoded secret keys
    - Missing security headers
  
  - **Lack of Security Architecture**
    - No input validation framework
    - Missing rate limiting
    - No logging or monitoring
  
  **Security Impact:**
  - Information disclosure through debug errors
  - Session hijacking via predictable keys
  - Missing defense-in-depth layers
  
  **Mitigation Applied:**
  - Security headers middleware in `secure/app.py:19-26`
  - Environment-based configuration
  - CSRF protection with Flask-WTF
  - Proper error handling
</Accordion>

**Severity:** Medium  
**CVSS Score:** 6.5  
**Affected Code:** Application configuration

---

### A07:2021 - Identification and Authentication Failures

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **Insecure Session Management**
    - Weak session configuration
    - No session timeout
    - Session tokens not rotated
  
  - **Weak Authentication**
    - No password complexity requirements
    - No account lockout mechanism
    - Predictable session identifiers
  
  **Security Impact:**
  - Session hijacking
  - Credential stuffing attacks
  - Brute force attacks
  
  **Mitigation Applied:**
  - Secure session configuration in `secure/app.py:67`
  - Password length validation
  - Proper session cleanup on logout
  - Input sanitization and trimming
</Accordion>

**Severity:** High  
**CVSS Score:** 7.5  
**Affected Code:** Session and authentication logic

---

### A03:2021 - Cross-Site Scripting (XSS)

<Accordion title="Coverage in This Project">
  **Vulnerabilities Demonstrated:**
  
  - **Reflected XSS** - `vulnerable/app.py:82-86`
    - User input from URL parameters rendered without sanitization
    - JavaScript code executed in victim's browser
    - No Content Security Policy
  
  **Attack Vectors Shown:**
  ```html
  <!-- Alert Box -->
  <script>alert('XSS')</script>
  
  <!-- Cookie Theft -->
  <script>alert(document.cookie)</script>
  
  <!-- DOM Manipulation -->
  <script>document.body.innerHTML='HACKED'</script>
  
  <!-- Event Handler -->
  <img src=x onerror="alert('XSS')">
  ```
  
  **Security Impact:**
  - Session token theft
  - Phishing attacks
  - Keylogging
  - Website defacement
  
  **Mitigation Applied:**
  - Output escaping with MarkupSafe in `secure/app.py:136`
  - Content Security Policy headers
  - Input validation and sanitization
  - Auto-escaping in Jinja2 templates
</Accordion>

**Severity:** High  
**CVSS Score:** 7.5  
**Affected Code:** Dashboard message display
