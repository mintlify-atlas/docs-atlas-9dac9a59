<Warning>

</Warning>

### Route: Dashboard (XSS)

Source: `vulnerable/app.py:75-87`

```python
@app.route('/dashboard')
def dashboard():
    if 'user_id' not in session:
        flash('Debes iniciar sesión', 'warning')
        return redirect('/login')
    
    # VULNERABLE: XSS
    message = request.args.get('message', '')
    
    return render_template('dashboard.html', 
                         username=session.get('username'),
                         message=message,
                         role=session.get('role'))
```

<Warning>
  **XSS Vulnerability**:
  - Unescaped user input passed to template
  - Combined with `{{ message|safe }}` in template, allows script injection
  - No Content Security Policy headers
</Warning>

### Route: Profile (IDOR)

Source: `vulnerable/app.py:89-110`

```python
@app.route('/profile')
def profile():
    if 'user_id' not in session:
        flash('Debes iniciar sesión', 'warning')
        return redirect('/login')
    
    user_id = request.args.get('id', session['user_id'])
    
    connection = create_connection()
    cursor = connection.cursor() 
    
    try:
        cursor.execute(f"SELECT * FROM users WHERE id = {user_id}")
        user = cursor.fetchone()
    except sqlite3.Error as e:
        flash(f'Error: {str(e)}', 'danger')
        user = None
    finally:
        cursor.close()
        connection.close()
    
    return render_template('profile.html', user=user)
```

<Warning>
  **Multiple Vulnerabilities**:
  
  1. **IDOR (Insecure Direct Object Reference)** - Any user can view any profile
  2. **SQL Injection** - User ID concatenated into query
  3. **No Authorization** - No permission checks
  4. **Password Exposure** - Returns all user fields including password
</Warning>

#### Exploitation Example

```bash

GET /profile?id=1

# View all users via SQL injection

```

### Database Setup

Source: `vulnerable/database.py:16-46`

```python
def setup_database():
    conn = sqlite3.connect(get_db_path())
    cursor = conn.cursor()
    
    cursor.execute("""
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            username TEXT NOT NULL UNIQUE,
            password TEXT NOT NULL,
            email TEXT,
            role TEXT DEFAULT 'user',
            created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
        )
    """)
    
    try:
        cursor.execute("""
            INSERT INTO users (username, password, email, role) 
            VALUES (?, ?, ?, ?)
        """, ('admin', 'admin123', 'admin@test.com', 'admin'))
        
        cursor.execute("""
            INSERT INTO users (username, password, email) 
            VALUES (?, ?, ?)
        """, ('usuario', 'password123', 'user@test.com'))
    except sqlite3.IntegrityError:
        pass
    
    conn.commit()
    conn.close()
```

<Warning>
  **Database Security Issues**:
  - **Plaintext Passwords** - Default users have plaintext passwords
  - **Weak Default Passwords** - 'admin123' and 'password123'
  - **No Indexes** - Performance issues at scale
  - **No Password Column Constraints** - No minimum length requirements
</Warning>

## Summary of Vulnerabilities

<AccordionGroup>
  <Accordion title="SQL Injection" icon="database">
    **Affected Routes**: `/login`, `/profile`
    
    User input concatenated directly into SQL queries using f-strings, allowing attackers to:
    - Bypass authentication
    - Extract database contents
    - Modify or delete data
    - Execute arbitrary SQL commands
  </Accordion>
  
  <Accordion title="Cross-Site Scripting (XSS)" icon="code">
    **Affected Routes**: `/dashboard`
    
    Unescaped user input rendered in templates with `|safe` filter, allowing attackers to:
    - Inject malicious JavaScript
    - Steal session cookies
    - Perform actions on behalf of users
    - Deface the application
  </Accordion>
  
  <Accordion title="Insecure Direct Object Reference (IDOR)" icon="lock-open">
    **Affected Routes**: `/profile`
    
    No authorization checks on user ID parameter, allowing:
    - Viewing other users' profiles
    - Accessing sensitive information
    - Privilege escalation
  </Accordion>
  
  <Accordion title="Plaintext Password Storage" icon="key">
    **Affected**: Database, `/register`, `/login`
    
    Passwords stored and compared in plaintext:
    - Database breach exposes all passwords
    - No protection against rainbow tables
    - Violates security best practices
  </Accordion>
  
  <Accordion title="Hardcoded Secrets" icon="eye">
    **Affected**: `app.secret_key`
    
    Secret key hardcoded in source:
    - Visible in version control
    - Same across all environments
    - Compromises session security
  </Accordion>
  
  <Accordion title="Missing Security Headers" icon="shield">
    **Affected**: All routes
    
    No security headers configured:
    - No CSRF protection
    - No XSS protection headers
    - No Content Security Policy
    - No HTTPS enforcement
  </Accordion>
</AccordionGroup>
